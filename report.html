1.1, XHTML5

This webpage developed by XHTML5 + CSS3. The main technology solutions were learnt from an online tutorial Except the comment html5 features, this report will highlight below features:

A) Mobile devices consideration.

Mobile devices were supported by meta tag which learnt from an online post. The layout also managed by css, that will be illustrated in css section.

B) Favicon.

Favicon is a browser sensitive feature. The solution of this project was learnt from http://en.wikipedia.org/wiki/Favicon “.icon” picture is recommended format. However the picture generated by GIMP was not recognized. So this project used a png file for favicon.

C) All involved tabs:

Header,nav,ul,article,footer,aside,pre.
- header is a header of a document or a section.
- footer is a footer of html section.
- nav is navigation.
- ul is unodered list.
- article is a container for post or comment
- aside is a container for content around main content.
- pre is a container which preserves both spaces and line breaks.

All items displayed on this webpage were created by html tags but the layout was managed by css.

1.2, CSS3

CSS plays the main role of this project, which added styles to web pages. Several technology solutions will be highlighted in this report:

A) CSS inheritance

CSS inheritance be commonly recognized as sub tags inherit styles from parent tags (and other situations). However, in this project, corner radius was boardly used style for all containers. Copy these styles into all css sectors can be a solution, but there will be a lot of copies in css files. if a value need to be changed, that change need to be implemented into all places. Therefore a CSS inheritance feature is needed: 
.rounded_corners {  
   -moz-border-radius: 8px;  
   -webkit-border-radius: 8px;  
   border-radius: 8px;
}
#header {  
  .rounded_corners;
}
#footer {
  .rounded_corners;
}
However, after research, there are three solutions:
Solution 1: Creating a base style and appling it to html tags. However, this is not the real inheritance. 
<article class='content rounded_corners'>
Solution 2: Overriding. Buy this solution, the base styles were only able to be shared to one sub css sector. 
.topcontent {
	border-radius : 5px;
	-moz-border-radius : 5px;
	-webkit-border-radius : 5px;
}

.topcontent {
	background-color: #FFF;
}
After the consideration of all solutions, this project didn't adopt any of them. The final solution is making copies.

B) Mobile

"@media" offers mobile style control ability. 
@media only screen and (max-width: 480px)  and (min-width: 150px){ }
All styles in @media will be activated when the screen smaller 480px. All css are used for overriding the default styles. For example, below styles are used for making navigations to display vertically, but still keep the default styles, such as coner radius. 
@media only screen and (max-width: 480px)  and (min-width: 150px){
	.body{
		width: 90%;
		font-size:95%;
	}
	.mainHeader img{
		width : 100%;
	}
	.mainHeader nav{
		height:160px;
	}
	.mainHeader nav ul {
		/*height:100%;*/
		padding-left:0;
	}
	.mainHeader nav ul li{
		width:100%;
		text-align:center;
	}
	.mainHeader nav a:link,.mainHeader nav a:visited{
		padding:10px 25px;
		display:block;
	}
}

C) Multi styles for links(for different status)

“a:link” for style of a normal link, “a:visited” for clicked links, “a:hover” for links mouse over. For example: 
.mainHeader nav a:link,.mainHeader nav a:visited{
	color:#FFF;
	display: inline-block;/*http://www.w3schools.com/cssref/pr_class_display.asp*/ /*http://designshack.net/articles/css/whats-the-deal-with-display-inline-block/*/
	padding:10px 25px;
	height: 20px;
}

.mainHeader nav a:hover, .mainHeader nav a:active,
.mainHeader nav .active a:link, .mainHeader nav .active a:visited{
	background-color:#CF5C3F;	
	text-shadow: none;
}

D)Units

Px is a commonly used units on web page design. Px depends on resolution, it provides good across platforms ability. But it is not recommended to use, because the text then doesn't scale when the visitor changes the default text size. em and percentage are widely used units in this project. 1em = 100% = current font size. By default, 1em equals 16px. To avoid use px directly, “font-size : 87.5%;” is commonly used way to control the font size. If “font-size = 87.5%”, then 1em = 16px * 87.5%.

1.3.2,Validation

Parsleyjs was adopted in this project. Parsleyjs is a very simple javascript library, To add this ability into a html web page, developers can simply add "data-parsley-validate" into "form" tag.
"required" is used for configurating a compulsion field.
"type="email"" is used for setting a email field, this field will be validated as an email address.
<form id="demo-form" data-parsley-validate>
